Making Of

This page shows how we planned our project design by using canva along with the justification for using CSS and JavaScript.

Header and Footer

By using footer.js and navbar.js we made the header and footer come from shared templates so we avoid repeating them. The header also changes based on the user status and plan and if no one is signed in then it only shows Home, Story, and Making Of.

Home Page

On this page, clicker the register button shows a form using login.js (removes d-none from form and hide button). if user registered the welcomeCard appears to great the user and let him chose a plan. Using plan.js the chosen plan decides access: Basic can't use Nutrition, while Premium unlocks everything.

Register

The inputs are validated in login.js (name length longer than 3, email format, and 8 digit phone). If everything is correct and valid then the data is saved in localStorage.

Products

Once the user is signed in, he will be able to see the products that we offer. If he clicks on one, it will be added to the cart using localStorage below and then he can click Submit. An image showing more info will be sent to the phone number he entered, handled by products.js.

Facilities

There is no JavaScript for this page except for checker.js that prevents users who didn't sign from accessing it.

Nutrition

In this page we wanted to use nutrition.js file that calculate the users required nutrition based on weight, height, age, and gender. also we check the users plan, and if its not allowed, an error will appear like the one shown above.

Story

We don't use any additional JavaScript files here. This page is only meant to show our story and the difficulties we faced.

Justification for colors + ErrorPage

We used purple to represent the sporty vibe while black for a cooler modern look and white as a clean neutral balance. We also created an error page handled by Error.js, that appears when someone tries to access the site without signing in.
